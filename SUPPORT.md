## Community Support

### GitHub Discussions

For questions, ideas, and community interaction, use [GitHub Discussions](https://github.com/olaflaitinen/promethium/discussions):

| Category | Purpose |
|----------|---------|
| Q&A | Ask questions and get answers from the community |
| Ideas | Propose new features or improvements |
| Show and Tell | Share your projects using Promethium |
| General | General discussion about seismic processing and the project |

### Best Practices for Asking Questions

To receive effective help, please:

1. **Search first**: Check existing discussions and documentation.
2. **Provide context**: Include your environment, versions, and configuration.
3. **Be specific**: Describe what you expected versus what happened.
4. **Include examples**: Provide minimal reproducible examples when possible.
5. **Format code**: Use Markdown code blocks for code and logs.

## Issue Reporting

### Bug Reports

For confirmed bugs, open an issue on [GitHub Issues](https://github.com/olaflaitinen/promethium/issues).

**Required Information:**

```markdown

- Promethium version:
- Python version:
- Operating system:
- Deployment method (Docker/local):

### Description
[Clear description of the bug]

### Steps to Reproduce
1. [First step]
2. [Second step]
3. [...]

### Expected Behavior
[What should happen]

### Actual Behavior
[What actually happens]

### Logs/Screenshots
[Relevant error messages or screenshots]
```

### Security Issues

**Do not report security vulnerabilities through public issues.**

See [SECURITY.md](SECURITY.md) for reporting security vulnerabilities.

## Feature Requests

We welcome feature requests through [GitHub Discussions](https://github.com/olaflaitinen/Promethium/discussions) in the Ideas category.

When proposing a feature:

1. **Describe the problem**: What challenge does this feature address?
2. **Describe your solution**: How would you like it to work?
3. **Provide use cases**: Real-world scenarios where this would help.
4. **Consider alternatives**: What other approaches did you consider?

Feature requests with clear use cases and community support are prioritized.

## Response Expectations

Promethium is maintained by volunteers. Response times depend on maintainer availability and issue complexity.

### Expected Response Times

| Channel | Initial Response | Resolution |
|---------|------------------|------------|
| Security issues | 48 hours | Priority handling |
| Bug reports | 1 week | Varies by severity |
| Feature requests | 2 weeks | Based on roadmap |
| General questions | 1-2 weeks | Community-dependent |

### How to Escalate

If you have not received a response within the expected timeframe:

1. Add a polite comment to your issue or discussion.
2. Ensure your report includes all required information.
3. Consider if the question might be better suited for a different channel.
